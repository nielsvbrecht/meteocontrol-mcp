# Product Guidelines: MCP Server for MeteoControl VCOM API v2

## Prose and Communication Style
- **Technical and Data-First:** Prioritize precision and accuracy in all data-related communication. Use objective and technical language that reflects the reliability of the underlying solar array telemetry.
- **Directness:** Aim for clarity and avoid unnecessary conversational filler. Ensure that the most critical data is always presented first.
- **Consistent Terminology:** Maintain strict consistency with the terminology used in the MeteoControl VCOM API v2 documentation.

## Accessibility Principles
- **Screen Reader Ready:** All responses generated by the MCP server must be formatted to be easily interpreted by screen readers, ensuring full accessibility for visually impaired users.
- **Simple and Intuitive Navigation:** Design the interaction flow to be as straightforward as possible, minimizing the cognitive load required to find and retrieve specific data.
- **Standard Formatting:** Use standardized Markdown structures (headers, lists, tables) to provide a clear and predictable document hierarchy for assistive technologies.

## Design and User Experience (UX) Goals
- **Efficiency-First:** Optimize the MCP server for rapid data extraction and low-latency interaction, enabling users to get answers as quickly as possible.
- **Clarity and Precision:** Ensure that complex data points (e.g., energy generation, inverter efficiency, alert statuses) are presented in a clear and unambiguous manner.
- **Ease of Onboarding:** Design the server so that new system operators and asset managers can begin interacting with the VCOM API v2 and getting value immediately with minimal training.

## Error Handling and Troubleshooting
- **User-Friendly Error Messages:** Provide error messages that are clear, concise, and easy to understand, even for users who may not have deep technical knowledge of the API internals.
- **Actionable Guidance:** When an error occurs (e.g., a failed API request or missing data), provide helpful advice on how the user can resolve the issue or where they can find more information.
- **Security Awareness:** Ensure that error messages do not expose sensitive system information or internal API secrets.
